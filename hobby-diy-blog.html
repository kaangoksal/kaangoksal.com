Playing With Quadrocopters!

7/10/2015

0 Comments

 
I have been playing with quadrotors for 4 months now. What I mean by playing is autonomous flights and building quadrotor frames by scratch with 3D printers, carbon plates and tubes. We had numerous crashes due to hardware failure, electromagnetic interference, vibration and loose propellers - not to forget poor mission planning. 
I played with many different frames. Tube arms are really painful, you have to align them very well and then glue, or drill them to have a solid and trustworthy frame, otherwise you can see your quad start to spin wildly. Vibration can loosen up the bolts and change the angle of the motors. If you crash with a carbonfiber frame, then you have to replace the broken parts which would take up some time not mentioning the cost.  
After all that R&D proccess, I figured out that the best frame is the one that you designed and have quick access to! If you have a 3D printer and some decent solidworks skills you can create your very own frame in days!. So I designed my own quadrocopter frame. It took about 27 hours. I'm still improving it. There are many things to consider while designing your own frame. I will make a list in my following blog posts.

IOT ﻿Climate Sensor! 

3/1/2015

0 Comments

 
I made a IoT Climate probe with ESP8266 and gy-65 pressure and temperature sensor. It measures the temperature, calculates the altitude and logs them to a cloud server that I'm currently hosting on my home server.  I also planned to print the status of the device to the OLED. I was planning to print the temperature, pressure, battery status, wifi properties. However Arduino Nano ran out of RAM! Look! 
This is caused by the Oled, since it is a 128*64 display, the library it uses creates a 128*64 size array, and stores it in the RAM. I have two choices, one to use TWO arduinos! One for the LCD and the other one is for the wifi and climate sensor. The second is use EEPROM, or add another RAM IC kind of thing. This is a perfect learning opportunity! I didn't use EEPROM before. However right now I like to see the results, I'll probably stick with two arduinos and then later on I'll optimize the code to decrease it into one arduino.
The picture on the left is a test where I print the Temperature, pressure and the altitude to Oled, The wifi is off. I dont use any libraries with ESP8266, real men dont use libraries... No just kidding, I didn't like any of the libraries available.

ESP8266 TCP COMMUNICATION

2/27/2015

0 Comments

 
I continued to experiement on ESP8266. I discovered how to send TCP packets to a Server on my local area network. The results were pretty satisfactory. I used a java applet called SocketTest, and the code below to send the packets from ESP8266. The SocketTest applet is not asynchronous so when the client (arduino -esp8266) looses connection, you have to disconnect the client from the server manually and re-connect again. The SocketServer isn't perfect, but it works fine for testing. 
#include <SoftwareSerial.h>
#define SSID "xxxxxxxxx"      //your wifi ssid here
#define PASS "xxxxxxxxx"   //your wifi wep key here
#define DST_IP "192.168.2.8" //baidu.com
#define HOST "kaangoksal.com"
 
SoftwareSerial dbgSerial(11, 10); // RX, TX
 
char myChar;
 
void setup()
{
  pinMode(9, OUTPUT);
  pinMode(13, OUTPUT);
 
  //blink led13 to indicate power up
  for(int i = 0; i<15; i++)
  {
   digitalWrite(13,HIGH);
   delay(50);
   digitalWrite(13,LOW);
   delay(50);
  }
 
  // Open serial communications for WiFi module:
  dbgSerial.begin(9600);
  // Set time to wait for response strings to be found
  dbgSerial.setTimeout(5000);
 
  //Open software serial for debugging - must use serial to usb (FTDI) adapter
  //dbgSerial.begin(19200); //can't be faster than 19200 for softserial
  Serial.begin(38400);   //38400 softserial works fine for me
 
  Serial.println("ESP8266 Demo");
  delay(100);
 
  //test if the module is ready
  dbgSerial.println("AT+RST");    
  if(dbgSerial.find("ready"))
  {
    Serial.println("Module is ready");
    delay(1000);
    //connect to the wifi
    boolean connected=false;
    for(int i=0;i<5;i++)
    {
      if(connectWiFi())
      {
        connected = true;
        break;
      }
    }
    if (!connected)
    {
      //die
      while(1);
    }
 
    delay(5000);
    //set the single connection mode
    dbgSerial.println("AT+CIPMUX=0");
  }
  else
  {
    Serial.println("Module didn't respond.");
    delay(100);
   
    //serial loop mode for diag
    while(1){
      while (Serial.available()) {
        myChar = Serial.read();
        dbgSerial.print(myChar);
      }
 
      while (dbgSerial.available()) {
        myChar = dbgSerial.read();
        delay(25);
        Serial.print(myChar);
      }
    }
  }
}
 
void loop()
{
  String cmd = "AT+CIP\"TCP\",\"";
  cmd += DST_IP;
  cmd += "\",21";
  dbgSerial.println(cmd);
  Serial.println(cmd);
  if(dbgSerial.find("Error")) return;
  cmd = "Transmitting data well and clear! ";
  cmd += HOST;
  cmd += "\r\n\r\n";
 
  dbgSerial.print("AT+CIPSEND=");
  dbgSerial.println(cmd.length());
  if(dbgSerial.find(">"))
  {
    Serial.print(">");
  }
  else
  {
    dbgSerial.println("AT+CIPCLOSE");
    Serial.println("connect timeout");
    delay(1000);
    return;
  }
 
  dbgSerial.print(cmd);
 
  delay(500);
  //Serial.find("+IPD");
  while (dbgSerial.available())
  {
   char c = dbgSerial.read();
   Serial.write(c);
   if(c=='\r') Serial.print('\n');
   delay(50);
  }
 
 Serial.println();
 Serial.println("====");
 delay(1000);
}
 
 
boolean connectWiFi()
{
  dbgSerial.println("AT+CWMODE=1");
  String cmd="AT+CWJAP=\"";
  cmd+=SSID;
  cmd+="\",\"";
  cmd+=PASS;
  cmd+="\"";
  Serial.println(cmd);
  dbgSerial.println(cmd);
  delay(2000);
  if(dbgSerial.find("OK"))
  {
    Serial.println("OK, Connected to WiFi.");
    return true;
  }
  else
  {
    Serial.println("Can not connect to the WiFi.");
    return false;
  }            
}

5$ Wi-fi,  ESP8266 , Arduino IoT

1/18/2015

0 Comments

 
i bought ESP8266  wifi modules from Aliexpress. They are extremely cheap compared to 33$ TI CC3000 Wi Fi shield. So far i couldn't make the chips work, i couldn't get any response. There is a high chance that i might have burned one. öhöm öhöm... MODULES ARE RATED 3.3V and a guy at a random forum insisted that he didn't have any problems with 5v logic and 3.3v supply from Arduino, not mentioning that tx and rx pins are 3.3v, so i tried to use digital pins for uart. the chip didn't produce any smoke or so. But the dealer I bought them from says that THEY ARE NOT 5V Tolerant, one might be burned up! 

I realized that Arduino can't supply enough current for the module, it draws 70mA (max of arduino 5v, 3.3v pins) .

i created an alternative voltage supply circuit to supply the required power. I did this with a voltage divider from a 5V 1A power supply. It didn't work out. Tomorrow I'm going to use an Arduino Mega Power Supply.  

Acording to the posts over the internet the wifi chip has a range of 200m with pcb antenna and all tcp, ip stuff built in. it supports wpa/ wep, can be an access point??????
i will report my progress!
The picture at the right is from SparkFun.com 
https://learn.sparkfun.com/tutorials/using-the-logic-level-converter

You can find more detailed info at their page! 
I made this module work after a week of trial-error. Happily I didn't burn any hardware. Things I used.

LOGIC LEVEL CONVERTER (BI-DIRECTIONAL)
3.3V External Voltage Supply (AMS1117) 
2 Arduino Nanos
9V battery

How did I make the modules work? 
First the logic converter must be hooked up correctly. RX -TX  ------- RX-TX connections are important. Both of the RX TX of the Wifi module is connected to TX s of the logic converter. As you see in the picture above, TX sections of the logic converter is bi directional, but RX sections are not! You need to wire up all the four pins (HV, GND LV, GND) (maybe not all the gnds) to make it work. If you don't connect LV funny things will occur. 
Second step was finding the correct baud rate which turned out to be 9600, many online forums suggested that the default was 115200, however things get updated fast! I bought the chips from Aliexpress! After setting the baud rate to 9600 everything seemed fine! However the module started to restart from time to time. When I added external 3.3V power supply It all went well! Mama mia! #cheapwifi
﻿I used the code that I found at http://www.seeedstudio.com/wiki/WiFi_Serial_Transceiver_Module to test my circuit and so. However I tweaked the code acording to my hookup and 9600 baud rate with software serial. Here is the edited version.

EXPERIMENTING WITH PRUSA I3

8/30/2014

0 Comments

 
I have been using prusa i3 for a week now. I had dozens of problems over the week, fillament types, bed leveling, Prusa's double stepper z axis calibration, Slic3r configuration and some Repetier Host bugs.

What I've learned
ABS melts at 250-260
ABS need some tweaking on the print bed such as acetone abs solution, kaptan tape
PLA 220-230
no need for a heated bed but I use 50C
PLA melts at lower temperature


 
I'm using Repetier-Host 1.0.2 with my Melzi board. Right now I'm happy with the performance of these both but I still have issues. 

The temperature is fluxuating. This effects the quality of the small parts. Especially If you're printing 3mm wide cylinders.
